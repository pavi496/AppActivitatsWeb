Descripció i objectius del projecte

Descripció i Objectius del Projecte Aquest projecte té com a objectiu principal el desenvolupament d'una aplicació de gestió per a un gimnàs. Mitjançant l'ús de la tecnologia JavaFX per a la interfície gràfica i MySQL com a sistema gestor de bases de dades, s'intenta proporcionar als usuaris una plataforma versàtil i eficient per a la gestió de les seves activitats físiques. Entre els objectius específics es troben la possibilitat de registrar-se com a usuari, reservar classes dirigides, consultar horaris, i gestionar els pagaments i les subscripcions. Cronograma del Projecte El cronograma del projecte s'ha establert i documentat detalladament en una fulla de càlcul accessible a través de Google Drive. Aquesta planificació inclou una llista exhaustiva de tasques, amb els corresponents terminis i assignacions de recursos, per assegurar un desenvolupament eficaç i ordenat del projecte. Disseny de Base de Dades Model Conceptual (Diagrama E-R) El model conceptual de la base de dades està dissenyat per reflectir de manera clara i concisa les relacions entre les diferents entitats del sistema. Les principals taules inclouen "Usuaris", "Activitats", "Horaris" i "Reserves", connectades mitjançant claus primàries i foranes per establir les relacions adequades entre elles. Model Lògic-Relacional El model lògic-relacional proporciona una visió més detallada de les taules de la base de dades, definint les claus primàries, les claus foranes i les restriccions d'integritat necessàries per mantenir la coherència del sistema. Disseny Físic: Script DDL SQL i SQL per a proves (insercions) Per a la implementació de la base de dades, s'ha elaborat un script DDL SQL que permet la creació de les taules i la definició de les restriccions de la base de dades. A més, s'han realitzat insercions de dades de prova per a garantir el correcte funcionament del sistema. UML Diagrama de Classes El diagrama de classes proporciona una representació visual de l'estructura del projecte, mostrant les relacions entre els diferents components com ara models, vistes i controladors. Això permet una comprensió més clara de l'arquitectura de la solució i facilita el desenvolupament i la mantenció del codi. Diagrama d'Ús El diagrama d'ús identifica i descriu les interaccions entre els actors i el sistema, definint els casos d'ús principals i secundaris de l'aplicació. Aquesta representació gràfica ajuda a comprendre millor el comportament del sistema i a validar els requeriments funcionals. Manual Bàsic d'Ús de l'Aplicació Informàtica del Projecte El manual d'ús proporciona una guia detallada sobre com utilitzar l'aplicació, des de l'inici de sessió fins a la realització de diferents tasques com ara la reserva de classes, la consulta d'horaris i la gestió del perfil d'usuari. S'inclouen captures de pantalla i exemples pràctics per facilitar la comprensió i l'ús de l'aplicació. Incidents i Problemes Es detallen els problemes i les dificultats trobades durant el desenvolupament del projecte, així com les solucions implementades per superar-los. Aquests incidents poden incloure errors en el codi, incompatibilitats amb les versions de les llibreries utilitzades o dificultats en la integració de components. Documentació Tota la documentació del projecte es troba disponible en un lloc web dedicat, ja sigui en una plataforma com Google Sites o GitHub. Aquesta documentació està organitzada de manera estructurada per facilitar l'accés i la consulta dels diferents apartats com ara els requeriments, el disseny, les proves i les instruccions d'ús.
